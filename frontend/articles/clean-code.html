Clean Code: Writing Code for Humans

"Any fool can write code that a computer can understand. Good programmers write code that humans can understand." ‚Äî Martin Fowler

Introduction

Clean code is code that is easy to read, easy to understand, and easy to maintain. It's not just about making code work; it's about making it sustainable for the long term.

Meaningful Names

Names should reveal intent. Avoid using generic names like `data`, `info`, or `item`.

JavaScript 
// Bad
const d = new Date(); // elapsed time in days

// Good
const elapsedTimeInDays = new Date();

Functions

Functions should be small and do one thing (Single Responsibility Principle).

JavaScript 
// Bad
function processUser(user) {
  validateUser(user);
  saveToDatabase(user);
  sendWelcomeEmail(user);
}

// Good
function registerUser(user) {
  if (isValid(user)) {
    saveUser(user);
    notifyUser(user);
  }
}

Comments

Don't comment bad code‚Äîrewrite it. Comments should be used to explain "why", not "what".

SOLID Principles

The SOLID principles are five design principles intended to make software designs more understandable, flexible, and maintainable.

  • S: Single Responsibility Principle
  • O: Open-Closed Principle
  • L: Liskov Substitution Principle
  • I: Interface Segregation Principle
  • D: Dependency Inversion Principle

DRY and KISS

  • DRY (Don't Repeat Yourself): Every piece of knowledge must have a single, unambiguous, authoritative representation within a system.
  • KISS (Keep It Simple, Stupid): Most systems work best if they are kept simple rather than made complicated.

Formatting

Consistent formatting is essential for readability. Use tools like Prettier or ESLint to automate this.
