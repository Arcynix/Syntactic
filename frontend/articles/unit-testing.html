Unit Testing Best Practices

Testing üìÖ Updated: December 2024 ‚è±Ô∏è 18 min read

Unit testing is a software testing method by which individual units of source code are tested to determine whether they are fit for use.

Introduction

A unit is the smallest testable part of any software. It usually has one or a few inputs and usually a single output.

Why Unit Test?

  • Find Bugs Early: Catch issues before they reach production.
  • Facilitate Change: Refactor code with confidence.
  • Documentation: Tests serve as documentation for how code should behave.
  • Design Improvement: Writing tests often leads to better code architecture.

The AAA Pattern

The AAA (Arrange, Act, Assert) pattern is a common way of writing unit tests.

  • Arrange: Set up the conditions for the test.
  • Act: Execute the code being tested.
  • Assert: Verify that the result is what you expected.
JavaScript (Jest) 
test('adds 1 + 2 to equal 3', () => {
  // Arrange
  const a = 1;
  const b = 2;

  // Act
  const result = add(a, b);

  // Assert
  expect(result).toBe(3);
});

F.I.R.S.T. Principles

Effective unit tests should follow the F.I.R.S.T. principles:

  • Fast: Tests should run quickly.
  • Independent: Tests should not depend on each other.
  • Repeatable: Tests should yield the same results every time.
  • Self-Validating: Tests should clearly pass or fail.
  • Thorough: Tests should cover all edge cases.

Mocking and Dependency Injection

When a unit has dependencies (like a database or an API), you should "mock" those dependencies to isolate the unit being tested.

JavaScript (Jest) 
// Mocking a module
jest.mock('./api');
import { fetchData } from './api';

test('gets data successfully', async () => {
  fetchData.mockResolvedValue({ id: 1, name: 'Test' });
  const data = await getUser(1);
  expect(data.name).toBe('Test');
});

Code Coverage

Code coverage is a measure used to describe the degree to which the source code of a program is executed when a particular test suite runs.

Note: 100% coverage doesn't mean 0% bugs, but it's a good indicator of test thoroughness.
