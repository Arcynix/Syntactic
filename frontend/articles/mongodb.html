MongoDB & NoSQL Databases

Database üìÖ Updated: December 2024 ‚è±Ô∏è 18 min read

MongoDB is a source-available cross-platform document-oriented database program. Classified as a NoSQL database program, MongoDB uses JSON-like documents with optional schemas.

What is NoSQL?

NoSQL (Not Only SQL) databases provide a mechanism for storage and retrieval of data that is modeled in means other than the tabular relations used in relational databases.

Core Concepts

MongoDB's data model is based on documents and collections.

Documents

A document is a record in a MongoDB collection and the basic unit of data in MongoDB. Documents are similar to JSON objects but exist in a format called BSON (Binary JSON).

JSON 
{
  "_id": "507f1f77bcf86cd799439011",
  "name": "John Doe",
  "age": 30,
  "email": "john@example.com",
  "address": {
    "street": "123 Main St",
    "city": "New York"
  },
  "tags": ["developer", "mongodb"]
}

Collections

A collection is a group of MongoDB documents. It is the equivalent of an RDBMS table. A collection exists within a single database.

CRUD Operations

Basic operations for interacting with MongoDB using the shell or a driver.

JavaScript 
// Create
db.users.insertOne({ name: "Alice", age: 25 });

// Read
db.users.find({ age: { $gt: 20 } });

// Update
db.users.updateOne(
  { name: "Alice" },
  { $set: { age: 26 } }
);

// Delete
db.users.deleteOne({ name: "Alice" });

Indexing

Indexes support the efficient execution of queries in MongoDB. Without indexes, MongoDB must perform a collection scan, i.e. scan every document in a collection, to select those documents that match the query statement.

JavaScript 
// Create an index on the email field
db.users.createIndex({ email: 1 }, { unique: true });

When to Use MongoDB?

  • Flexible Schema: When your data model changes frequently.
  • Scalability: When you need to scale horizontally (sharding).
  • Real-time Analytics: Fast writes and powerful aggregation.
  • Content Management: Storing diverse types of content.
