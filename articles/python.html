Python is a high-level, interpreted programming language known for its simplicity and readability. Created by Guido van Rossum and first released in 1991, Python has become one of the most popular programming languages in the world.

Introduction

Python emphasizes code readability with its notable use of significant indentation. Its syntax allows programmers to express concepts in fewer lines of code than would be possible in languages such as C++ or Java.

Python supports multiple programming paradigms, including:

  • Object-oriented programming - Classes and objects
  • Functional programming - First-class functions and lambda expressions
  • Procedural programming - Structured programming with functions

Basic Syntax

Python's syntax is designed to be readable and straightforward. Here are fundamental concepts:

Variables and Data Types

Python is dynamically typed, meaning you don't need to declare variable types explicitly:

Python 
# Variable assignment
name = "Alice"
age = 30
height = 5.6
is_student = True

# Multiple assignment
x, y, z = 1, 2, 3

# Type conversion
num_str = str(123)  # "123"
num_int = int("456")  # 456

Control Flow

Python uses indentation to define code blocks instead of braces:

Python 
# If statements
if age >= 18:
    print("Adult")
elif age >= 13:
    print("Teenager")
else:
    print("Child")

# For loops
for i in range(5):
    print(i)

# While loops
count = 0
while count < 5:
    print(count)
    count += 1

Data Structures

Python provides several built-in data structures that are incredibly useful:

Lists

Lists are ordered, mutable collections that can contain items of different types:

Python 
# Creating lists
fruits = ["apple", "banana", "cherry"]
numbers = [1, 2, 3, 4, 5]
mixed = [1, "two", 3.0, True]

# List methods
fruits.append("orange")     # Add item
fruits.remove("banana")     # Remove item
fruits.sort()              # Sort list
length = len(fruits)       # Get length

# List comprehension
squares = [x**2 for x in range(10)]

Dictionaries

Dictionaries store key-value pairs and provide fast lookup:

Python 
# Creating dictionaries
person = {
    "name": "Alice",
    "age": 30,
    "city": "New York"
}

# Accessing values
name = person["name"]
age = person.get("age", 0)

# Dictionary methods
person["job"] = "Engineer"  # Add/update
person.update({"country": "USA"})
keys = person.keys()
values = person.values()

Functions

Functions in Python are defined using the `def` keyword:

Python 
# Basic function
def greet(name):
    return f"Hello, {name}!"

# Function with default parameters
def power(base, exponent=2):
    return base ** exponent

# Function with multiple return values
def get_stats(numbers):
    return min(numbers), max(numbers), sum(numbers)

# Lambda functions
square = lambda x: x**2
add = lambda a, b: a + b

Object-Oriented Programming

Python supports OOP with classes and objects:

Python 
# Defining a class
class Dog:
    def __init__(self, name, breed):
        self.name = name
        self.breed = breed
    
    def bark(self):
        return f"{self.name} says Woof!"
    
    def __str__(self):
        return f"{self.name} is a {self.breed}"

# Creating objects
my_dog = Dog("Buddy", "Golden Retriever")
print(my_dog.bark())
print(my_dog)

# Inheritance
class Puppy(Dog):
    def __init__(self, name, breed, age_months):
        super().__init__(name, breed)
        self.age_months = age_months

Best Practices

Follow these guidelines for writing clean Python code:

  • Follow PEP 8 - Python's style guide for consistent code formatting
  • Use meaningful names - Variables and functions should be descriptive
  • Write docstrings - Document your functions and classes
  • Keep functions small - Each function should do one thing well
  • Use list comprehensions - When appropriate for readability
  • Handle exceptions - Use try-except blocks for error handling

Common Pitfalls

Avoid these common mistakes when learning Python:

  • Modifying lists while iterating over them
  • Using mutable default arguments in functions
  • Forgetting to close files (use context managers instead)
  • Not understanding the difference between `==` and `is`
  • Overusing global variables
