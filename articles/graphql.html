GraphQL: A Query Language for APIs

API üìÖ Updated: December 2024 ‚è±Ô∏è 20 min read

GraphQL is a query language for APIs and a runtime for fulfilling those queries with your existing data. GraphQL provides a complete and understandable description of the data in your API.

Introduction

Developed internally by Facebook in 2012 before being publicly released in 2015, GraphQL allows clients to define the structure of the data required, and exactly the same structure of data is returned from the server.

Key Concepts

GraphQL is based on a few core concepts that differentiate it from REST.

Schema

A GraphQL schema is at the center of any GraphQL server implementation. It describes the functionality available to the clients which connect to it.

Queries

A GraphQL query is used to read or fetch values. In REST, you would use a GET request.

GraphQL 
query {
  user(id: "123") {
    name
    email
    posts {
      title
    }
  }
}

Mutations

Mutations are used to write or post data. In REST, you would use POST, PUT, PATCH, or DELETE.

GraphQL 
mutation {
  createPost(title: "New Post", content: "Hello World") {
    id
    title
  }
}

Benefits of GraphQL

  • No Over-fetching or Under-fetching: Get exactly what you need and nothing more.
  • Strongly Typed: The schema defines the types and relationships.
  • Single Endpoint: All data is accessed through a single URL.
  • Introspection: Clients can query the schema itself for details about the API.

GraphQL vs REST

While REST is based on resources (URLs), GraphQL is based on a graph of data. REST often requires multiple requests to fetch related data, while GraphQL can fetch it all in one go.
