Git: Version Control Essentials

Tools üìÖ Updated: December 2024 ‚è±Ô∏è 15 min read

Git is a distributed version control system that tracks changes in any set of computer files, usually used for coordinating work among programmers collaboratively developing source code during software development.

Introduction

Created by Linus Torvalds in 2005 for development of the Linux kernel, Git has since become the most widely used version control system in the world.

Basic Commands

These are the most common commands you'll use in your daily workflow.

Bash 
# Initialize a new repository
git init

# Clone an existing repository
git clone https://github.com/user/repo.git

# Check status
git status

# Add changes to staging
git add .

# Commit changes
git commit -m "Your commit message"

# Push to remote
git push origin main

Branching and Merging

Branching allows you to diverge from the main line of development and continue to do work without affecting that main line.

  • git branch [name]: Create a new branch.
  • git checkout [name]: Switch to a branch.
  • git merge [name]: Merge a branch into the current one.

Collaboration

Git is designed for collaboration. You can pull changes from others and share your own.

  • git pull: Fetch and merge changes from remote.
  • git fetch: Download objects and refs from another repository.

Best Practices

  • Commit Often: Keep commits small and focused.
  • Write Good Commit Messages: Explain the "why", not just the "what".
  • Use Branches: Don't work directly on `main`.
  • Pull Before Push: Stay up to date with the remote repository.
